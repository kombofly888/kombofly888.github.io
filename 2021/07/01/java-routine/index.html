2.1 线程和进程

2.1.1 区别

2.1.1.1 进程
  • 概念
    1. 进程是指运行中的程序。当运行了一个程序，就启动了一个进程，操作系统会为该进程分配内存空间。
    2. 进程是程序的一次执行过程，或是正在运行的一个程序。是动态过程：有它自身的产生、存在和消亡的过程。
2.1.1.2 线程
  • 概念
    1. 线程是由进程创建的，是进程的一个实体。
    2. 一个进程可以拥有多个线程。
  • 单线程、多线程
    1. 单线程：同一个时刻，只允许执行一个线程
    2. 多线程：同一个时刻，可以执行多个线程
    3. 并发：同一个时刻，多个任务交替执行，造成一种貌似同时的错觉，简单来说，单核CPU实现的多任务就是并发。
    4. 并行：同一时刻，多个任务同时执行。多核CPU可以实现并行。
  • Java创建线程的两种方式
    1. 继承Thread类，重写run方法
      1. 当一个类继承了Thread类，该类就可以当线程使用
      2. 重写run方法，实现业务逻辑
    2. 实现Runnable接口，重写run方法
      1. 由于thread只能继承一个类的局限性，往往使用runnable接口
      2. 代理模式
      3. 更适合多个线程共享一个资源的情况
  • 线程机制
    1. 当运行代码时，等于启动了一个进程。main线程被创建。
    2. 当线程中使用start()， 创建一个子线程。
      • start() 启动线程，最终会执行该对象中的run方法
    3. 多线程交替执行，且当全部线程都执行完成，进程才结束
  • 线程常用方法
    1. setName
    2. getName
    3. start
    4. run
    5. setPriority
    6. getPriority
    7. sleep
    8. interrupt：中断线程，但没有真正的结束线程。一般用于中断正在休眠的线程
    9. yield： 线程礼让。让出CPU，不一定成功
    10. join：线程的插队
  • 用户线程和守护线程
    1. 用户线程： 也叫工作线程，当线程的任务执行完或通知方式结束
    2. 守护线程(Daemon)：一般是为工作线程服务的，当所有的用户线程结束，守护线程自动结束。eg, 垃圾回收机制
      • 使用方法： setDaemon(true)

2.1.3 同步Synchronized

  • 概念

    • 保证数据在任何时刻，最多有一个线程访问，以保证数据的完整性

  • 实现方法

    1. 同步代码块

    2. 放在方法声明中，表示整个方法为同步方法

  • 互斥锁

    1. 每个对象都对应于一个可称为 互斥锁 的标记，这个标记用来保证在任一时刻，只能由一个线程访问该对象
    2. 关键字 Synchronized 来于对象的互斥锁联系
    3. 局限性：导致程序的执行效率降低
    4. 同步方法(非静态的)的锁可以是this，也可以是其他对象
    5. 同步方法(静态的)的锁为当前类本身， 当前类.class
    6. 多个线程的锁必须是同一个对象

  • 释放锁

    1. 当前同步执行完成
    2. 遇到break，return
    3. 出现了未处理的Error或Exception

6.1.3.2 三大基本特征，原则
6.1.3.2.1 封装
  • 封装就是把抽象出来的数据（属性）和对数据的操作（方法）封装在一起，数据被保护在内部，外界只能调用被授权的操作（方法）来对数据进行操作。
  • 优点：
    1. 隐藏实现细节：只能用方法来传入或调出参数
    2. 可以对数据进行验证，保证数据安全合理
  • 实现：
    1. 属性私有化private
    2. 提供一个公共的(public) set方法，用于对属性判断并赋值
    3. 提供一个公共的(public) get方法，获取属性的值
6.1.3.2.2 继承

  • 继承概念: 当多个类中存在共有属性和方法，则从这些类中抽象出父类，让父类拥有共有的属性和方法，这样子类就不用重复定义这些属性和方法，减少了代码复用。

  • 继承细节：

    1. 子类继承了父类所有的属性和方法，但无法直接访问私有属性和方法
    2. 子类必须调用父类的构造器，完成父类的初始化
      • 子类会默认调用父类的无参构造器(存在隐藏语句 super())。若父类没有提供无参构造器，则子类必须调用super去指定使用父类的其他构造器
      • 若希望指定调用父类某个构造器，则显式的调用一下
    3. super在使用时，只能放在构造器第一行
      • super() 和 this() 都只能放第一行，因此两方法不能共存在一个构造器内
    4. Java 所有类都是 Object类
    5. 父类构造器的调用不限于直接父类，将一直向上追溯直至Object类
    6. 子类只能继承一个父类，单继承机制
    7. 不能滥用继承，需满足 is-a 逻辑关系

  • 继承的内存布局

    1. 加载类，优先加载超类，父类再本类
    2. 创建对象后，堆内分配空间，空间内依次存放超类，父类，子类
    3. 对象的引用指向对象地址
    4. 访问属性时，按照查找关系来返回信息
      • 子类存在该属性，并且可以访问，则返回；若不可以访问，报错
      • 子类不存在，查询父类是否存在
    5. 调用方法，
      • 先找本类，如果有，则调用。如果没有找父类

    继承本质详解

6.1.3.2.3 多态(Polymorphic)

​ 方法或对象具有多种形态，建立在封装和继承基础之上的。

  • 方法的多态：6.1.3.5.4重载和重写

  • 对象的多态（重点）

    1. 一个对象的编译类型和运行类型可以不一致

      ​ Animal animal = new cat();

      • Animal 编译类型， cat() 运行类型
      • 编译类型在定义对象时就确定了，不能改变
      • 运行类型可以改变
      • 编译类型看定义时 =号 左侧， 运行类型看 =号 右侧

    2. 对象调用方法时，由运行类型决定方法。

    3. 编译类型可以指向该类型的子类对象

  • 细节

    • 属性无法重写，访问属性看编译类型类
    • 调用方法看运行类型类

  • 应用

    • 多态数组
      • instanceof 比较运行类型的类是否相同
    • 多态参数
      • 其表现常在传参中发生，在形参列为父类，实参为子类

6.1.3.3 四个关键字
6.1.3.3.1 static 静态

  1. 类变量/静态变量

    是该类类所有对象共享，任何一个该类的对象去访问它时，取到的都是相同的值。

    • 可以通过 类名.类变量名 或者 对象名.类变量名 来访问
    • 在类加载的时候就生成了,即使没有创建对象，依然能够使用类变量
    • 类变量的生命周期是随类的加载开始，随类的消亡而摧毁
    • 同类加载一样，只会调用一次

  2. 类方法/静态方法

    • 当方法中不涉及任何和对象相关的成员，可以设计成静态方法，提高开发效率，eg. Math.sqrt()
    • 类方法和普通方法都是随着类的加载而加载，将结构信息存储在方法区
    • 类方法不允许使用和对象有关的关键字，比如this和super
    • 类方法中只能访问 静态变量或静态方法
    • 普通成员方法即可以访问非静态成员 也能访问静态成员
6.1.3.3.2 final

修饰类，属性，方法，局部变量。

  1. final 类： 类无法被继承，即不希望类被继承时，用final

    • 可以实例化

  2. final 属性：属性无法修改，即不希望类的某个属性值被修改

    • final修饰的属性(常量)要大写来命名，字间用下划线间隔

    • 在定义时必须赋值，且以后不能修改

      • 定义时赋值
      • 构造器内赋值
      • 代码块内赋值(若属性还是静态，则不能在构造器中赋值)

    • final 和 static 一起使用时，效率更高。若只用到了该种属性，且属性为基本数据类型和String，则不会导致类加载。(编译器常量）??

    • 当final 修饰的属性是直接数据类型时，数值不能修改

      当final修饰的属性是引用类型时，引用对象(地址)不能改变(但是对象内的属性等可变），eg: int[]

  3. final 方法：方法不能被重写，即不希望父类的某个方法被子类重写

    • 如果一个类已经是final类了，则没有必要用final修饰类中方法

  4. final 局部变量：固定局部变量，无法修改

6.1.3.3.3 this

即为当前对象，哪个对象调用，this就代表那个对象， 可以理解为对象内存在一个this属性指向当前对象

  • this关键字可以用来返回属性，方法，构造器
  • this 用于区分 当前类的属性和局部变量
  • 访问成员方法的语法: this.方法名(参数列表)
  • 访问构造器的语法：this(参数列表)， 且只能放在构造器内的第一条语句
  • this 不能在类定义的外部使用，只能在类定义的方法中使用
6.1.3.3.4 super

代表父类的引用，用于访问父类的属性，方法和构造器。

  • 当父类与子类有重名时，为了访问父类的成员，必须通过super。若没有重名，使用super、this、直接访问效果一样
  • 访问顺序，直接从父类开始，然后遵循继承访问规则
6.1.3.4 初始化
6.1.3.4.1类加载的三种情况

类只会加载一次

  • 初次创建该类的对象
  • 创建子类的对象，父类也会加载
  • 使用类的静态成员时(类变量，类方法)被调用
6.1.3.4.2 代码块

代码块，又称初始化块，属于类中的成员。类似于方法，但没有方法名，没有返回，没有参数，只有方法体。不用通过对象或类显示调用，而是加载时，或创建对象时隐式调用。

  • 静态代码块，即 static {}，对类进行初始化

    1. 在类加载时执行，同类加载一样，只会执行一次

    2. 调用时，优先级和静态属性初始化一样，按定义的顺序调用(谁在上面谁先访问)

    3. 静态代码块只能调用静态成员

  • 普通代码块/非静态代码块

    1. 在创建对象时执行，每创建一个对象，执行一次(被隐式调用)
    2. 发生在 构造器 之前（构造器最前面隐含了super()和调用普通代码块）
    3. 发生在 父类 之后
    4. 调用时，优先级同普通属性初始化一样，谁在上谁访问
    5. 普通代码块可以调用任意成员
6.1.3.4.3 初始化顺序 :warning:

​ 创建对象 ->

  1. 首次创建会加载类，若有继承，访问父类的静态代码块和静态属性

  2. 访问本类的静态代码块和静态属性初始化

​ 完成加载 ->

  1. 进入对象的构造器内
  2. 判断有无父类，若有， ->进入父类
  3. 访问父类的普通代码块和普通属性初始化
  4. 访问父类的构造器, -> 返回子类
  5. 访问子类的普通代码块和普通属性初始化
  6. 访问子类的构造器
6.1.3.5 属性和方法
6.1.3.5.1 方法调用机制
  • 调用方法时，会在栈中分配一块独立空间，该方法内的局部变量仅作用于空间内。当方法执行完最后一句时，空间将被摧毁，
6.1.3.5.2 方法传参机制

  • 基本数据类型的传参机制：

    ​ 传递的是值，形参的任何改变不影响实参。

    ​ 当传递参数为基本数据类型时，实参存储于栈中。执行方法时，会额外生产一个栈空间进行操作，实参的值传递给形参。形参将发生改变但不影响实参。

  • 引用数据类型的传参机制：

    ​ 当传递参数为引用数据类型时，堆中会提供一个地址用于方法执行，栈中的实参指向该地址。执行方法时，也会额外生产一个栈空间进行操作，但此时实参会把堆地址传递给形参。因此，若形参进行改变会对对象的属性造成变化。

  • 两道思考题

    Hint: 在方法里面创建对象不是明智之举

  • 克隆对象

6.1.3.5.3 构造方法

​ 没有返回值的方法，作用是便于初始化对象

​ 构造器的最前面隐含了 super() 和 调用普通代码块

  • 构造函数（构造器）

    • 没有返回值
    • 方法名必须和类名一致
    • 形参列表 和 成员方法一样规则
    • 构造器的调用由系统完成
    • 可以重载多个构造器

  • 默认构造函数

    • 存在默认构造器 方法名(){}
    • 一旦自定义了构造器，那么默认构造器就覆盖，可重载
6.1.3.5.4 重载和重写

  • 重载(Overload)

    ​ 同一个类中，当多个方法方法名同名，形参列表不同， 即为重载

    • 方法名相同
    • 形参列表不同（数量不同或形参类型不同或顺序不同）
    • 返回类型不影响

  • 重写(Override)

    当子类中存在一个方法，和父类有相同的方法名称，参数列表，返回类型时，子类这个方法付给了父类的方法

    • 方法名相同
    • 参数一样
    • 返回类型一样或者是父类返回类型的子类
    • 子类方法不能缩小父类方法的访问权限

  • 两者区别

名称 发生范围 方法名 形参列表 返回类型 修饰符
重载 同类 相同 数量、类型、顺序至少一个不同 无要求 无要求
重写 子类 相同 相同 相同或者是返回类型的子类 大于父类

6.1.5.1 List

  • 概念

    1. List集合类中元素有序、且可重复

    2. List集合中的每个元素都有其对应的顺序索引，即支持索引

    3. List的实现类有 ArrayList、 LinkedList 和Vector等

  • 常用方法

  • ArrayList

    1. ArrayList 可以加入null

    2. 是由数组来实现数据存储的

    3. 基本等同于Vector，但是ArrayList是线程不安全的，多线程情况下不建议使用

  • ArrayList的扩容机制

    1. 维护了一个Object类的数组elementData

    2. 当创建对象时，如果使用的是无参构造器，初始容量elementData容量为0，第一次添加，扩容为10。如需再次扩容，则扩容elementData为1.5倍

    3. 如果使用的是指定大小类型的构造器，则初始elementData容量为指定大小。如需扩容，则直接扩容elementData为1.5倍

  • Vector 类

    1. Vector底层也是一个对象数组

    2. Vector 是线程同步的，即线程安全，Vector类的操作方法带有synchronized

  • Vector 扩容机制

    1. 无参，默认10，满后按2倍扩容
    2. 有参，直接按两倍扩容

  • LinkedList 类

    1. 底层维护了一个双向链表
    2. 维护了两个属性 first 和 last，分别指向首节点和尾节点
    3. 每个节点(Node对象)，又维护了prev、next、item三个属性。通过prev指向前一个，通过next指向后一个。
    4. 线程不安全，没有实现同步
    5. 可以添加null

  • ArrayList和LinkedList比较

    1. 如果改查的操作比较多，选择ArrayList
    2. 如果增删的操作比较多，选择LinkedList
    3. 一般程序中，查询占比较高，大部分情况选择ArrayList
6.1.5.2 Set
  • 基本介绍
    1. 无序（添加和取出的顺序不一致），没有索引
    2. 不允许重复元素，最多包含一个null
    3. 实现类有：Hashset, TreeSet等

  • HashSet 类

    1. 实现了set接口
    2. 实际上是HashMap
    3. 数组+链表+红黑树的结合

  • HashSet 添加元素机制

    1. 添加一个元素时，先得到Hash值，在通过相应的转换，变成索引值
    2. 在存储数据表table中，查看这个索引位置是否已经存在元素
    3. 如果没有直接加入。如果有，调用equals比较，相同就放弃添加。不同则添加到最后。
    4. 当一条链表的元素个数达到8 或者table的大小大于等于64，则该链表转换成红黑树。

  • HashSet 扩容机制

    1. 第一次添加时，table数组扩容到16，加载因子时0.75

    2. 每添加一个元素，size会+1，当size到达临界值时16*0.75，就会扩容一倍到32。

  • LinkedHashSet

    1. LinkedHashSet 是 HashSet的子类

    2. 底层是一个LinkedHashMap， 底层维护了一个数组 + 双向链表

    3. 根据元素的hashcode值来决定元素的存储位置，同时用链表维护元素的次序，使得元素看起来是以插入顺序保存的

    4. 不允许重复的元素

  • TreeSet

    1. 使用无参构造器时，创建TreesSet，依然是无序的
    2. 想要排序，必须传入一个比较器，重载比较器的比较方法
    3. 根据底层源码比较器相关部分，当比较的内容不同时，会按照给定的顺序排序。当比较的内容相同时，该数据不会被放进树中。
6.1.5.3 Map - 很实用

  • Map接口实现类的特点

    1. 保存具有映射关系的数据：Key_Value
    2. Map中的 Key 和 Value 可以是任何引用类型的数据，会封装到HashMap$Node对象中
    3. Map中的 Key 不允许重复，原因和HashSet一样。当加入重复的key时，新的key对应的Value会替换原Value
    4. Map中的 Key 可以为 null，但只能存在一个null。Value可以存在任意数量的null。
    5. 通常用String类 作为 Map的 Key
    6. key和value之间存在单向一对一关系，即通过指定的key 总能找到对应的value

  • 常用方法

    方法 功能
    boolean contains(Object value) 确定哈希表内是否包含了给定对象
    boolean containsKey(Object key) 确定哈希表内是否包含了给定的关键词
    boolean isEmpty() 确定哈希表是否为空
    Object get(Object key) 获取对应关键字的对象
    Object put(Object key, Object value) 用给定的关键字把对象保存到哈希表中
    Object remove(Object key) 从哈希表中删除与给定关键字相对应的对象
    Void resize() 扩容
    getOrDefault(Object key, n)
    Object keySet()

  • Map遍历方式

  • HashMap ：

    1. HashMap 是 Map接口使用频率最高的实现类
    2. HashMap 是以 key-val对 的方式来存储数据
    3. key不能重复，value可以重复
    4. 添加相同的key，会覆盖原来的key-val（key不改变，val覆盖）
    5. 与HashSet一样，不保证映射顺序
    6. HashMap没有实现同步，线程不安全

  • HashMap 扩容机制

    1. 第一次添加时，table数组扩容到16，加载因子时0.75
    2. 每添加一个元素，size会+1，当size到达临界值时16*0.75，就会扩容一倍到32。
  • HashTable :
    1. 跟HashMap同层，以Key-val对 的方式来存储数据
    2. key，value不能是NULL
    3. 实现了同步，线程安全
    4. 底层有数组 Hashtable$eEntry[] 初始化大小为11
    5. 临界值 threshold 8 = 11 * 0.75
    6. 当存放数组数量大于临界值时，将会扩容，为原原先的两倍+1
    7. 效率没有HashMap高
  • Properties
    1. 继承了Hashtable类，并实现了Map接口
    2. 使用特点和Hashtable类似
    3. 用于从properties文件中加载数据到Properties类对象
  • TreeMap
    1. 同理TreeSet
    2. 加入相同的键时，key不替换，value替换
6.1.5.4 总结
  • 判断使用哪种集合
    1. 存储类型是单列还是双列
      1. 单列：Collection接口
        1. 允许重复：List接口
          1. 增删多：LinkedList（底层双向链表）
          2. 改查多：ArrayList（底层Object类的可变数组）
        2. 不允许重复：Set接口
          1. 无序：HashSet（底层HashMap)
          2. 有序：TreeSet（底层TreeMap）
          3. 插入和取出顺序一致：LinkedHashSet
      2. 双列：Map接口
        1. 键无序：HashMap（底层是 哈希表，数组+链表+红黑树）
        2. 键有序：TreeMap
        3. 键插入和取出顺序一致：LinkedHashMap
        4. 读取文件：Properties

6.1.8 范型

  • 概念
    1. 格式 interface {} class<K,V>{}

    3. 在类声明或实例化时只要指定好需要的具体的类型
  • 好处
    1. 编译时，检查添加元素的类型，提高了安全性
    2. 减少了类型转换的次数，提高效率
      • 不使用泛型时，放入到ArrayList会先转换成Object，取出时还需转换成相应的类
      • 使用泛型后，存入和取出直接用到是指定类，不需要类型转换
    3. 不在提示编译警告
  • 细节
    1. E具体的数据类型在定义对象的时候指定，即在编译期间，就确定E是什么类型
    2. 泛型只能放入引用类型，不能放入基本数据类型
    3. 没有给泛型指定类型时，默认 Object
  • 自定义泛型类
    1. 语法
      • class 类名<T,R…>{
      • T,R 等为泛型的标识符，一般单个大写
    2. 普通成员可以使用泛型
    3. 方法可以使用泛型
    4. 使用泛型的数组不能初始化
      • 不能确定类型，无法开辟空间
    5. 静态不能使用泛型。
      • 类加载时，对象还没有创建。
  • 自定义泛型接口
    1. 语法
      • interface 接口名 <T,R…> {}
    2. 静态成员不能使用泛型
      • 接口的成员都是静态
    3. 泛型接口的类型，在继承接口或实现接口时确定
    4. 没有指定类型，默认 Object
  • 自定义泛型方法
    1. 语法
      • 修饰符<T,R…> 返回类型 方法名（参数列表）{}
    2. 可以定义在普通类中，也可以定义在泛型类中
    3. 注意区分 使用泛型 和 泛型方法 的区别
      • 使用泛型：在泛型类里面，一个方法使用了类声明的泛型,public void hi(T t){}
      • 泛型方法：public<U,M> void eat(U u, M m){}
  • 泛型的继承和通配符
    1. 泛型不具备继承性
    2. : 支持任意泛型类型
    3. : 支持 x 类以及 x类的子类，规定了泛型的上限
    4. : 支持 x类以及x类的父类，规定了泛型的下限

  • 异常处理方式

    • try-catch-finally

      • 如果发生了异常，直接进入catch块，try块中异常后面的代码不会执行
      • 如果没有发生异常，顺序执行try代码块，catch代码块不执行
      • 如果try代码块可能有多个异常，可以使用多个catch 分别捕获不同异常，相应处理。要求子类异常写在前面，父类异常写在后面。

    • throws

      当一个方法中存在异常时，该方法可以用try-catch 或者 抛出异常 给上级方法。同理上级方法可处理异常或者抛出异常，直到main方法，再到JVM，此时JVM会输出异常信息并且退出程序。

      当程序员没有显示的用try-catch处理异常，代码默认throw异常。

      • throws 后面的异常类型可以是方法中产生的异常类型，也可以是它的父类。
      • 子类重写父类方法时，对抛出异常的规定：必须一致，或者是父类抛出异常的子类型
