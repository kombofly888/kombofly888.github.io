2.1.1.1 进程
  • 概念
    1. 进程是指运行中的程序。当运行了一个程序，就启动了一个进程，操作系统会为该进程分配内存空间。
    2. 进程是程序的一次执行过程，或是正在运行的一个程序。是动态过程：有它自身的产生、存在和消亡的过程。
2.1.1.2 线程
  • 概念
    1. 线程是由进程创建的，是进程的一个实体。
    2. 一个进程可以拥有多个线程。
  • 单线程、多线程
    1. 单线程：同一个时刻，只允许执行一个线程
    2. 多线程：同一个时刻，可以执行多个线程
    3. 并发：同一个时刻，多个任务交替执行，造成一种貌似同时的错觉，简单来说，单核CPU实现的多任务就是并发。
    4. 并行：同一时刻，多个任务同时执行。多核CPU可以实现并行。
  • Java创建线程的两种方式
    1. 继承Thread类，重写run方法
      1. 当一个类继承了Thread类，该类就可以当线程使用
      2. 重写run方法，实现业务逻辑
    2. 实现Runnable接口，重写run方法
      1. 由于thread只能继承一个类的局限性，往往使用runnable接口
      2. 代理模式
      3. 更适合多个线程共享一个资源的情况
  • 线程机制
    1. 当运行代码时，等于启动了一个进程。main线程被创建。
    2. 当线程中使用start()， 创建一个子线程。
      • start() 启动线程，最终会执行该对象中的run方法
    3. 多线程交替执行，且当全部线程都执行完成，进程才结束
  • 线程常用方法
    1. setName
    2. getName
    3. start
    4. run
    5. setPriority
    6. getPriority
    7. sleep
    8. interrupt：中断线程，但没有真正的结束线程。一般用于中断正在休眠的线程
    9. yield： 线程礼让。让出CPU，不一定成功
    10. join：线程的插队
  • 用户线程和守护线程
    1. 用户线程： 也叫工作线程，当线程的任务执行完或通知方式结束
    2. 守护线程(Daemon)：一般是为工作线程服务的，当所有的用户线程结束，守护线程自动结束。eg, 垃圾回收机制
      • 使用方法： setDaemon(true)

6.1.3.2.1 封装
  • 封装就是把抽象出来的数据（属性）和对数据的操作（方法）封装在一起，数据被保护在内部，外界只能调用被授权的操作（方法）来对数据进行操作。
  • 优点：
    1. 隐藏实现细节：只能用方法来传入或调出参数
    2. 可以对数据进行验证，保证数据安全合理
  • 实现：
    1. 属性私有化private
    2. 提供一个公共的(public) set方法，用于对属性判断并赋值
    3. 提供一个公共的(public) get方法，获取属性的值
6.1.3.2.2 继承

  • 继承概念: 当多个类中存在共有属性和方法，则从这些类中抽象出父类，让父类拥有共有的属性和方法，这样子类就不用重复定义这些属性和方法，减少了代码复用。

  • 继承细节：

    1. 子类继承了父类所有的属性和方法，但无法直接访问私有属性和方法
    2. 子类必须调用父类的构造器，完成父类的初始化
      • 子类会默认调用父类的无参构造器(存在隐藏语句 super())。若父类没有提供无参构造器，则子类必须调用super去指定使用父类的其他构造器
      • 若希望指定调用父类某个构造器，则显式的调用一下
    3. super在使用时，只能放在构造器第一行
      • super() 和 this() 都只能放第一行，因此两方法不能共存在一个构造器内
    4. Java 所有类都是 Object类
    5. 父类构造器的调用不限于直接父类，将一直向上追溯直至Object类
    6. 子类只能继承一个父类，单继承机制
    7. 不能滥用继承，需满足 is-a 逻辑关系

  • 继承的内存布局

    1. 加载类，优先加载超类，父类再本类
    2. 创建对象后，堆内分配空间，空间内依次存放超类，父类，子类
    3. 对象的引用指向对象地址
    4. 访问属性时，按照查找关系来返回信息
      • 子类存在该属性，并且可以访问，则返回；若不可以访问，报错
      • 子类不存在，查询父类是否存在
    5. 调用方法，
      • 先找本类，如果有，则调用。如果没有找父类

6.1.3.3.1 static 静态

  1. 类变量/静态变量

    是该类类所有对象共享，任何一个该类的对象去访问它时，取到的都是相同的值。

    • 可以通过 类名.类变量名 或者 对象名.类变量名 来访问
    • 在类加载的时候就生成了,即使没有创建对象，依然能够使用类变量
    • 类变量的生命周期是随类的加载开始，随类的消亡而摧毁
    • 同类加载一样，只会调用一次

  2. 类方法/静态方法

    • 当方法中不涉及任何和对象相关的成员，可以设计成静态方法，提高开发效率，eg. Math.sqrt()
    • 类方法和普通方法都是随着类的加载而加载，将结构信息存储在方法区
    • 类方法不允许使用和对象有关的关键字，比如this和super
    • 类方法中只能访问 静态变量或静态方法
    • 普通成员方法即可以访问非静态成员 也能访问静态成员
6.1.3.3.2 final

修饰类，属性，方法，局部变量。

  1. final 类： 类无法被继承，即不希望类被继承时，用final

    • 可以实例化

  2. final 属性：属性无法修改，即不希望类的某个属性值被修改

    • final修饰的属性(常量)要大写来命名，字间用下划线间隔

    • 在定义时必须赋值，且以后不能修改

      • 定义时赋值
      • 构造器内赋值
      • 代码块内赋值(若属性还是静态，则不能在构造器中赋值)

6.1.3.4.2 代码块

代码块，又称初始化块，属于类中的成员。类似于方法，但没有方法名，没有返回，没有参数，只有方法体。不用通过对象或类显示调用，而是加载时，或创建对象时隐式调用。

  • 静态代码块，即 static {}，对类进行初始化

    2. 调用时，优先级和静态属性初始化一样，按定义的顺序调用(谁在上面谁先访问)

    3. 静态代码块只能调用静态成员

  • 普通代码块/非静态代码块

    1. 在创建对象时执行，每创建一个对象，执行一次(被隐式调用)
    2. 发生在 构造器 之前（构造器最前面隐含了super()和调用普通代码块）
    3. 发生在 父类 之后
    4. 调用时，优先级同普通属性初始化一样，谁在上谁访问
    5. 普通代码块可以调用任意成员

6.1.3.5.2 方法传参机制

  • 基本数据类型的传参机制：

    ​ 传递的是值，形参的任何改变不影响实参。

    ​ 当传递参数为基本数据类型时，实参存储于栈中。执行方法时，会额外生产一个栈空间进行操作，实参的值传递给形参。形参将发生改变但不影响实参。

  • 引用数据类型的传参机制：

    ​ 当传递参数为引用数据类型时，堆中会提供一个地址用于方法执行，栈中的实参指向该地址。执行方法时，也会额外生产一个栈空间进行操作，但此时实参会把堆地址传递给形参。因此，若形参进行改变会对对象的属性造成变化。

    3. 头插法，当元素插入到相同的table序列中时，会插在链表的头部

  • 取模机制

    1. 在HashMap中，当要决定key对应的table下标时，会先对key进行hash运算，然后再让hash值 跟 table容量进行取模。取得的模值即为插入的下标。
    2. capacity必须为2的n次方，这样capacity-1 为 (n-1)个1。 让h 与 capacity-1 做与运算， 得出的结果会和 h 对 capacity取模的结果一致。
    3. 位运算的代价比求模运算小的多，能带来更高的性能。
  • HashTable :
    1. 跟HashMap同层，以Key-val对 的方式来存储数据
    2. key，value不能是NULL
    3. 实现了同步，线程安全
    4. 底层有数组 Hashtable$eEntry[] 初始化大小为11
    5. 临界值 threshold 8 = 11 * 0.75
    6. 当存放数组数量大于临界值时，将会扩容，为原原先的两倍+1
    7. 效率没有HashMap高
  • Properties
    1. 继承了Hashtable类，并实现了Map接口
    2. 使用特点和Hashtable类似
    3. 用于从properties文件中加载数据到Properties类对象
  • TreeMap
    1. 同理TreeSet
    2. 加入相同的键时，key不替换，value替换
6.1.5.4 总结
  • 判断使用哪种集合
    1. 存储类型是单列还是双列
      1. 单列：Collection接口
        1. 允许重复：List接口
          1. 增删多：LinkedList（底层双向链表）
          2. 改查多：ArrayList（底层Object类的可变数组）
        2. 不允许重复：Set接口
          1. 无序：HashSet（底层HashMap)
          2. 有序：TreeSet（底层TreeMap）
          3. 插入和取出顺序一致：LinkedHashSet
      2. 双列：Map接口
        1. 键无序：HashMap（底层是 哈希表，数组+链表+红黑树）
        2. 键有序：TreeMap
        3. 键插入和取出顺序一致：LinkedHashMap
        4. 读取文件：Properties

    • 如果没有发生异常，顺序执行try代码块，catch代码块不执行
    • 如果try代码块可能有多个异常，可以使用多个catch 分别捕获不同异常，相应处理。要求子类异常写在前面，父类异常写在后面。

  • throws

    当一个方法中存在异常时，该方法可以用try-catch 或者 抛出异常 给上级方法。同理上级方法可处理异常或者抛出异常，直到main方法，再到JVM，此时JVM会输出异常信息并且退出程序。

    当程序员没有显示的用try-catch处理异常，代码默认throw异常。

    • throws 后面的异常类型可以是方法中产生的异常类型，也可以是它的父类。
    • 子类重写父类方法时，对抛出异常的规定：必须一致，或者是父类抛出异常的子类型

  • 自定义异常

    • 一般，自定义异常继承RuntimeException。这样就可以使用默认的处理机制

  • throw 和 throws 区别

    意义 位置 后面跟的东西
    throws 异常处理的一种方式 方法声明处 异常类型
    throw 手动生成异常对象的关键字 方法体中 异常对象

7.3.2 隔离级别

  1. 多个连接开启各自事务操作数据库中数据时，数据库系统要负责隔离操作，以保证各个连接在获取数据时的准确性。

  2. 不考虑隔离性，会有以下问题：

    1. 脏读：当一个事务读取另一个事务尚未提交的修改时，产生脏读
    2. 不可重复读：同一查询在同一事务中多次进行，由于其他提交事务所所做的修改uo删除，每次返回不同的结果集，此时会产生不可重复读
    3. 幻读：同一查询在同一事务所中多次进行，由于其他提交事务所做的插入操作，每次返回不同的结果集，此时发生幻读
