4. 容器功能

4.1 组件添加

@Configuration

  • SpringBoot中通过配置类，即config，来把bean注入到容器中

  • @Configuration告诉SpringBoot这是一个配置类 == 配置文件

    • 在Configuration中，有一个重要功能proxyBeanMethods，代理bean方法，默认为true，

      该功能能决定内部的bean是否要被代理来巩固bean生命周期表现。比如即使在用户代码中被直接调取了bean方法，也能返回一个单例实例。

      Specify whether @Bean methods should get proxied in order to enforce bean lifecycle behavior, e.g. to return shared singleton bean instances even in case of direct @Bean method calls in user code.

      当值为True时，当用户调用组件注册方法，会由内部代理来指向该类方法，并返回一个单例实例。被称作“Full Mode”,往往用在需要组件依赖时。该特质要求配置类满足CGLIB子类，并且类中的方法不能时final。

      当值为False时，用户会直接调用该类。多次调用，会导致返回的对象引用不一致。这也被叫做“Lite Mode”，轻量级操作，节约了时间。It is therefore behaviorally equivalent to removing the @Configuration stereotype.

      • 最佳实战

      • 配置 类组件之间无依赖关系用Lite模式加速容器启动过程，减少判断

      • 配置类组件之间有依赖关系，方法会被调用得到之前单实例组件，用Full模式

  • 给容器添加组件，返回类型就是组件类型。返回的值，就是容器中的实例。

    • 组件id有两种方法，第一种默认方法名，第二张在@Bean(value=””)中添加名字

  • 配置类里面使用@Bean标注在方法上给容器注册组件，默认也是单实例的

    • 配置类本身也是组件: 可以看到Configuration被注解了@Component的

@Import

给容器中自动创建出给定类型的组件，默认组件的名字就是全类名

@Conditional

条件装配：满足Conditional指定的条件，则进行组件注入

可以写在方法上或者TYPE上

其下有许多不同条件判断的继承注解

  • @ConditionalOnBean：可以根据给定的bean名字或者类型来判断是否要注入组件，可用于组件依赖

4.3 配置绑定

SpringBoot允许通过注解读取到properties文件中的内容并封装到JavaBean中

  • @ConfigurationProperties

    第一种方法，在实体类上直接加@ConfigurationProperties，value/prefix填入前缀名。注意实体类必须是从容器中的组件，@Component。

  • @EnableConfigurationProperties + @ConfigurationProperties

    方法二，在配置类中加上@EnableConfigurationProperties，括号内填入想要绑定的实体类名。并且实体类要加上@ConfigurationProperties

    @EnableConfigurationProperties 有两个功能：1.开启相关类的配置绑定功能。2.把该类自动注测到容器中
